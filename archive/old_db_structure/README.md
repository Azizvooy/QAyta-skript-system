# АРХИВ: Устаревшая структура БД

Эта папка содержит файлы, которые использовали старую структуру базы данных (fiksa_records, call_history_112).

## Дата архивирования
08.01.2026

## Архивированные файлы

### Импорт данных
- **import_call_history.py** - Импорт данных в старую таблицу call_history_112
  - Заменен на: `scripts/database/db_import.py`

### Анализ и сопоставление
- **show_data.py** - Просмотр данных из старых таблиц
  - Теперь используйте представления: v_fixations_full, v_applications_full
  
- **match_test.py** - Тестирование сопоставления данных
  - Не требуется: новая БД использует foreign keys
  
- **match_applications.py** - Сопоставление заявок и фиксаций
  - Не требуется: автоматическое через v_applications_full

## Новая структура

### Таблицы
- `operators` - Операторы (справочник)
- `services` - Службы (справочник)
- `regions` - Регионы (справочник)
- `applications` - Заявки (основная)
- `fixations` - Фиксации/обзвоны (основная)
- `operation_logs` - Логи операций
- `daily_statistics` - Статистика

### Представления (Views)
- `v_applications_full` - Полная информация о заявках с объединением
- `v_fixations_full` - Полная информация о фиксациях с объединением
- `v_operator_stats` - Статистика операторов

### Управление новой БД
Используйте:
- **УПРАВЛЕНИЕ_БД.bat** - Интерактивное меню управления БД
- `scripts/database/db_manager.py` - Python интерфейс
- `scripts/database/db_schema.py` - Создание схемы
- `scripts/database/db_import.py` - Импорт данных
- `scripts/database/db_reports.py` - Генерация отчетов

## Не удаляйте эту папку
Файлы сохранены для справки и возможности восстановления логики, если понадобится.
